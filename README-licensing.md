# مسیر — راه‌اندازی سیستم لینک اختصاصی و پنل مدیریت

این قابلیت به اپ اضافه شده: هر مشتری فقط با یک **لینک اختصاصی + یک کد فعال‌سازی** که تو بهش می‌دی می‌تونه وارد بشه، و اون لینک فقط روی **یک دستگاه** قابل فعال‌سازیه — اگه کسی همون لینک رو جای دیگه باز کنه و کد رو هم بزنه، رد می‌شه.

⚠️ **این یعنی اپ دیگه صد‌درصد آفلاین/بدون سرور نیست.** برای بار اول که یک لینک فعال می‌شه، نیاز به اینترنت و یک بک‌اند رایگان (Supabase) داره. بعد از فعال‌سازی اولیه، همون‌طور که قبلاً بود، بدون اینترنت هم کار می‌کنه.

## مرحله ۱ — ساخت پروژه در Supabase
1. برو به [supabase.com](https://supabase.com) و یک حساب رایگان بساز.
2. **New Project** بزن، یک اسم و رمز دیتابیس انتخاب کن (رمز رو یادت باشه، بعداً لازم نمی‌شه ولی گمش نکن).
3. چند دقیقه صبر کن تا پروژه ساخته بشه.

## مرحله ۲ — اجرای اسکریپت دیتابیس
1. توی پروژه، از منوی سمت چپ **SQL Editor** رو باز کن.
2. محتوای فایل `supabase-schema.sql` (همراه همین پکیج) رو کامل کپی و پیست کن.
3. **Run** بزن. باید بدون خطا تموم بشه.

## مرحله ۳ — ساخت حساب ادمین (خودت)
1. از منوی سمت چپ برو به **Authentication → Users**.
2. **Add user** بزن، ایمیل و رمز خودت رو وارد کن (این همون حسابیه که با اون وارد پنل مدیریت می‌شی).
3. توجه: این کار رو فقط برای خودت انجام بده؛ مشتری‌ها هیچ‌وقت اینجا حساب نمی‌سازن.

## مرحله ۴ — گرفتن کلیدهای اتصال
1. برو به **Project Settings → API**.
2. دو مقدار زیر رو کپی کن:
   - **Project URL**
   - **anon public key**

## مرحله ۵ — وارد کردن کلیدها توی فایل‌ها
هم توی `index.html` و هم توی `admin.html`، این بخش رو پیدا کن و مقادیرش رو جایگزین کن:
```js
const LICENSE_CONFIG = {
  SUPABASE_URL: "https://YOUR-PROJECT.supabase.co",
  SUPABASE_ANON_KEY: "YOUR-ANON-KEY"
};
```
همچنین توی `admin.html` این خط رو با آدرس واقعی سایتت (همون که توی README اصلی ساختی) جایگزین کن:
```js
const APP_BASE_URL = "https://USERNAME.github.io/masir-app/";
```

## مرحله ۶ — آپلود به گیت‌هاب
مثل قبل، همه‌ی فایل‌ها رو (شامل `admin.html` جدید) توی همون ریپازیتوری Upload → Commit کن:
- `index.html`
- `admin.html`
- `manifest.json`
- `service-worker.js`
- `icon-192.png`
- `icon-512.png`

## نحوه‌ی استفاده روزمره
1. برو به `https://USERNAME.github.io/masir-app/admin.html` و با ایمیل/رمزی که ساختی وارد شو.
2. اسم مشتری رو (اختیاری) بزن و **«+ ساخت لایسنس جدید»** رو بزن.
3. یک **لینک** و یک **کد ۶ رقمی** بهت نشون داده می‌شه — این کد فقط همین یک‌بار نمایش داده می‌شه، حتماً همون‌جا کپی/برای مشتری ارسال کن.
4. لینک رو به مشتری بده. اون لینک رو باز می‌کنه، کد رو وارد می‌کنه، و اپ براش فعال می‌شه — فقط روی همون یک گوشی.
5. اگه بعداً مشتری گوشیش عوض شد یا کش مرورگرش پاک شد و دیگه اپ رو نمی‌بینه: برو توی پنل، روی **«ریست دستگاه»** بزن؛ همون کد قبلی دوباره یک‌بار دیگه (روی دستگاه جدید) قابل استفاده می‌شه.
6. اگه خواستی دسترسی یک مشتری رو کامل قطع کنی: **«باطل کردن»** رو بزن — دیگه هیچ‌وقت با اون لینک/کد وارد نمی‌شه.

## محدودیت‌های مهم (صادقانه)
- «قفل به دستگاه» یعنی یک شناسه تصادفی که توی مرورگر همون گوشی ذخیره می‌شه — قوی و کافیه برای جلوگیری از فوروارد کردن لینک، ولی اگه مشتری کش مرورگرش رو پاک کنه یا اپ رو کامل حذف/نصب کنه، نیاز به «ریست دستگاه» از طرف تو داره (بالا توضیح داده شد).
- پلن رایگان Supabase برای شروع و رشد کاملاً کافیه؛ اگه تعداد مشتری‌ها خیلی زیاد شد (چند ده‌هزار به بالا)، ممکنه لازم باشه به یک پلن پولی‌ ارزون ارتقا بدی — چیزی که خودِ Supabase موقعش بهت اطلاع می‌ده.
- کد فعال‌سازی و ایمیل/رمز ادمین رو جایی امن نگه دار؛ هرکس رمز ادمین رو داشته باشه می‌تونه لایسنس بسازه یا باطل کنه.
